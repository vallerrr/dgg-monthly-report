# DGG Monthly Report (GitHub Pages)

Static dashboard hosted at https://vallerrr.github.io/dgg-monthly-report/

- No Python/runtime required for viewers.
- Displays **multiple months** via a month selector dropdown.
- Each month's data is a separate `YYYY-MM.json` file in `data/`.
- `data/index.json` lists all available months and is auto-regenerated by the pipeline.
- Includes interactive detail views: predicted trend, averaged FB trend, and current-month raw FB daily trend.

## Monthly update (from `dgg_pipeline`)

The pipeline writes directly into this repo's `data/` directory. From the `dgg_pipeline` root:

**Export + commit + push in one command:**

```bash
bash update_monthly_report_pages.sh --push
```

**Export only (no push):**

```bash
bash update_monthly_report_pages.sh
```

**Optional month override:**

```bash
bash update_monthly_report_pages.sh --year 2026 --month 3 --push
```

**Optional detail payload controls:**

```bash
bash update_monthly_report_pages.sh --data-type both --max-detail-pairs 500 --push
```

- `--data-type`: `fb_key` (default), `tessellated`, or `both`
- `--max-detail-pairs`: how many GID+outcome detail series to include per month

GitHub Actions redeploys automatically on every push to `main`.

## Data layout

```
data/
  index.json        ← auto-generated list of available months (do not edit manually)
  2026-03.json      ← payload for March 2026
  2026-02.json      ← payload for February 2026
  ...
```

`index.json` is regenerated by `export_static_payload.py` on every pipeline run. It lists months
newest-first so the website defaults to the latest month.

## GitHub Pages setup

In the GitHub repo settings:

- `Settings` → `Pages` → Source: `GitHub Actions`

The workflow at `.github/workflows/deploy-pages.yml` handles deployment automatically.
